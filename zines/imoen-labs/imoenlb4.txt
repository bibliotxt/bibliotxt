ReSpoNderÉ prEgunTas BásicAs A modo De GuÍa:

*PaRa Ke SiRve el NumeRo IP?
Una dirección o número IP es la forma que tiene TCP/IP de identificar cada
host que se encuentra conectado a una red El numero IP está formado poR
CuaTro cIFras, de uno a tres digitos cada una, separadas por puntos; por
ejemplo, 192.223.93.3 identifica a un unico host conectado a la red.

(NOTa: EsTa exPlicacion repito, NO pretende ser explicacion tecnica sino
otorgar un mero panorama del mundo de los  datagramas y paquetes...)

*Ke función cumple cada octeto?

Kada grupo de números recibe el nombre de octeto, y cumple una funcion
especial: algunos representasn el numero de red ( numero que identifica a
toda la red), y otros representan el numero de host (cada host tiene un
numero distinto). Esta asignacion de octetos depende del tipo de red que
se esté utilizando:

-ReD CLasE A : el primer octeto representa el numero  de red, mientras ke
los restantes tres octeto corresponden a numeros de Host.

-ReD CLaSe B : los dos primeros octetos son numeros de red y los dos ultimos
son numeros de host.

-ReD ClAsE C : los tres primeros octetos representan el número de red y el
cuarto octeto al numero de host.

Lo ke nosotros vemos como "octeto" son numeros decimales pero se trata en
realidad de un numero binario de ocho digitos o bits ( de ahi el nombre de
"octeto"). Siguiendo el ejemplo anterior:

              192                223              93                3 
        ************************ES EN REALIDAD...**************************
           11000000           11011111         01011101          00000011

De este modo, el máximo valor que puede tomar cada octeto es 255 (11111111) y
el minimo es 0 (00000000).

*Exiten otros números similares al IP?

Si, existe otro numero muy importante llamada "subnet mask"
(mascara de subred) que tiene el mismo formato que el IP (cuatro octetos).
Mediante él, se puede determinar si un host pertenece o no a una red, porke
permite identificar el numero de red y el numero de host de un IP.
Para ello, se realiza la operacion logica AND bit bit en todos los octetos
(una operación utilizada en programacion y demas, pero no se hagan problema
en entenderla, es muy  basicamente una "comparacion" de datos)

VeaMos un EjemPlo para explicarnos MejoR:
Tomemos el caso de una red de CLAsE C, entonces, el subnet mas será
255.255.255.0 (si fuera clase B seria: 255.255.0.0; en cambia si fuera
Clase A seria 255.0.0.0)
Tambien propongamos por caso que tenemos dos hosts; el host origen
con IP 192.223.93.40 y el host destino con IP 194.20.90.4. Keremos determinar
se el host destino pertenece a la red o no.

VeaMoS...

 11000000       11011111        01011101        00101000     --> IP origen
AND   11111111       11111111        11111111        00000000     --> Subnet Mask
--------------------------------------------------------                
11000000       11011111        01011101        00000000

 
11000010       00010100        01011010        00000100     --> IP destino
AND   11111111       11111111        11111111        00000000     --> Subnet Mask
--------------------------------------------------------
11000010       00010100        01011010        00000000

En ambos casos, el resultado es el numero de red del IP. Como los numeros de
red son distintos, entonces, los hosts pertencen a redes completamente
distintas. Si el host destino estuviera en la misma red, los resultados de
los AND hubieran sido iguales.


*Ke es la MAC address?

La MAC address es un numero hexadecimal que representa la direccion fisica de
un placa de red instalada en una computadora. Este numero es grabado
directamente en un chip de la misma por los fabricantes de la placa.
Todas las placas de red tienen MAC address diferentes ( o al menos
teoricamente...)

*De ke se tratan los mensajes broadcast?

Un mensaje Broadcast es un  mensaje enviado por un host sin un host destino
en particular: lo reciben todas las computadoras de la red.
POr EjeMplO, el protocolo ARP (explicado mas adelante) envía un mensaje
broadcast cuando se debe resolver un numero IP a una MAC address.
En otras palabras, dado un numero IP, se debe determinar la MAC address de
la placa de red del host qu posee dicho IP.

En ke consiste TCP/IP?
TCP/IP es una "suite" de protocolos que se utiliza en Internet para establecer
una comunicacion entre dos computadoras. Tambien es muy frecuente encontrar
redes "privadas"  (o no tanto :-)) que utilizan este conjunto de protocolos,
en este caso son las comunmente llamadas "intraNeTs" o LANs (KOmo las ke
hacen KuANdO JueGan AL QUAKE!! ONE FRAG lEFT...).
TCP/IP trabaja dentro del modelo de comunicacion cliente-servidor.
El modelo TCP/IP basado en el modelo OSI, incluye cuatro capas o layers
fundamentales (en realidad no hay un estandard para las capas (como si para
los modelos OSI, pero aki nombro las mas GeNeRaLes):

-APPLICATION LAYER: A traves de esta capa, las aplciaciones y programas tienen
acceso a la red. Cada version de  TCP/IP tendrá un protocolo para trabajar
con nombres de red y otro para trabajar con puertos de conexion ( un  cliente
de FTP es un ejemplo de este casO). En la version de MOcosOfT, el protocolo
NetBIOS es el usado para nombres de red, y Windows Sockets es la interface
de programacion que controla los puertos de conexion.

-TRANSPORT LAYER, o capa de transporte: Esta capa es la responsable de
establecer la comunicacion entre dos hosts. Los protocolos que actuan en el
Transport Layer son:

#TCP- Transmission Control Protocol: este protocolo "garantiza" la entrega
de los paquetes de datos que se envian a traves de la red, por lo tanto,
nos brinda una comunicación confiable. Esto se logra gracias a que TCP
necesita recibir una confirmacion del host destino de que efectivamente
recibió el paquete enviado, antes de enviar el siguiente (el sistema de
control de transferencia de datos de TCP se llama "sliding windows")

#UDP- User Datagram Protocol: este protocolo No garantiza la entrega correcta
de los paquetes como TCP. Ni siquiera  necesita tener una conexion establecida
entre los hosts, ya que envia los paquetes y no espera confirmacion alguna
del host destino. El control de la entrega de los paquetes queda a cargo de
la aplicacion que usa este protocolo.

UDP se usa para enviar paquetes mas pequeños que los que son enviados poR TCP.

#INTERNET LAYER, o capa de internet: Esta capa tiene a su cargo funciones
tales como el direccionamiento y ruteo de paquetes enviados a traves de la
red. En el Internet Layer podemos encontrar los siguientes protocolos:
 
  -/ IP . Internet Protocol: se encarga del direccionamiento y ruteo de los
  paquetes. No necesita de una conexion directa por lo que no garantiza la
  recepcion correcta de los paquetes: la responsabilidad de la entrega exitosa
  recae sobre las capas superiores.

  -/ ARP . Address Resolution Protocol: este protocolo realiza la resolución
  de un numero IP a una MAC address.

Para ello, envia un broadcast con el IP a resolver, y el hots que posee dicho
IP responde con un mensaje directo  (no broadcast), llamado "ARP reply",
enviando la informacion requerida. Toda comunicacion entre PCs de una red se
realiza a traves de la MAC address.

  -/ ICMP . Internet COntrol Message Protocol: es el protocolo encargado de
  reportar los errores que se producen al transmitir informacion (el mismo ke
  utilizan algunos nukes saturando el canal de reportes...)

-NETWORK INTERFACE LAYER, o capa de interfase de red: Es la parte de mas bajo
nivel del modelo TCP/IP. Esta capa se encarga de "poner" los paquetes que van
a enviarSe Direktamente en el cable, y de "LeVantar" del cable los paquetes
que llegan para pasarselos a las capas superiores.

*Ke es el TTL (Time to Live) de un paquete?
Indica el "tiempo de vida" del paquete enviado. Este valo disminuye, por
ejemple, cuando el paquete pasa por un default gateway (se explica mas abajao).
Cuando el TTL alcanza el valor cero, el paquete se descarta.

*Ke es un datagrama?
Es el paquetes de datos minimo que puede ser enviado a traves de la red desde
una computadorea hacia otra. Contiene solo encabezados, por eso su tamaño es
de apenas unos bytes.
Cuando un usuario hace un "ping" a un host (comando para determinar si dicho
activo esta activo o no, y para otras cosas...), se envian varios datagramas
al host destino y se cuenta su tiempo de respuesta.

*Ke es el default Gateway?

Es un aparato que sirve para conectar dos redes que se encuentren en lugares
geograficos distintos, o dos redes de distinta arquitectura (ethernet y token
ring, por ejemplo). Default gateway es sinonimo de lo que mas comunmente
conocemos como "router" en las comunicaciones. (NOtA: asi es como lo entiendo
yo pero en realidad no estoy muy seguro de si es completamente verdad, son
bienvenidos todos los aportes, sugerencias y criticas.)

Nota de imoen:  un gateway se encarga de unir dos redes con protocolos
distintos es decir una usa el TCP/IP y la otra el IPX/SPX el gateway se
encarga de realizar las conversiones necesaias para q se entiendan.


*Kuales son los comandos básicos de TCP/IP?
Ahora veremos algunos comandos que se utilizan comunmente en las redes TCP/IP.
Lo ideal seria usarlos en una ReD con un sistema operativo orientado a Redes,
como NT o Unix y sus variantes, pero tambien son funcionales en los demas
sistemas operativos ke son hoy dia lamentablemente estandards
(como Windows 9x o Me)

#~/ ping
Sirve para determinar si un host responde o no (puede ser local o remoto).
La sintaxis basica es "ping IP", pero pueden ver las variantes de la sintaxis
tecleando solo "ping". Este comando envia una serie de datagramas a IP
indicado y calcula el tiempo que tarda dicho host en responder
(si es ke responde).

Nota de imoen: Ya hicimos un articulo sobre el poder del ping
en un numero anterior ale a leerlo ! :)

#~/ ipconfig
Este es otro comando que permite ver la configuracion de IPs de la red a la
ke estamos conectados. Al igual que  ping, se puede usar estando conectados
a internet simplemente tipeandolo desde una ventana de DOS.
Proporciona informacion sobre nuestro numero IP, subnet mask e IP del
default gateway. Usando "ipconfig /all", obtendremos informacion adicional
como los IPs del DNS de nuestro servidor, MAC address de nuestra placa de red,
etc.

#~/ arp
Se escribimos "arp -a" en una ventana de DOS, podremos examinar lo que se
conoce como "ARP caché", que contiene los ultimos IPs resueltos por nuestro
host.

Proxys?
=======
En un principio los proxys fueron inventados para acelerar las conexiones a
Internet. Asi es como funcionan:
estas intentando conectar a un servidor en la otra punta del planeta. Tu
peticiones HTTP son enviadas por tu servidor proxy, ke esta localizado en
las oficinas de tu ISP, ke estan mucho mas proximas a ti ke ese servidor tan
lejano. El proxy primero chekea si uno de sus usuarios ha accedido a ese
website anteriormente. Si es asi, tendra una copia del mismo en alguna parte
de sus servers. Entonces el servidor proxy empieza la conexion solo para
chekear si su version del website no ha sido actualizada y solo necesita
mirar el tamaño del archivo. Si tiene la ultima version, te la mandara,
en vez de tener ke mandarte al servidor lejano, eso acelera la conexion.
Si no, el mismo descargara los archivos pedidos y entonces te los mandara.
Pero los proxys tambien se usan para ocultarte mientras navegas por la web,
porke soportan todas las peticiones HTTP por ti.
La mayoria de las veces tu ISP tiene un proxy. Llama y preguntales. Pero los
problemas con los accesos proxy de tu ISP son:
1. Algunos ISPs no tienen proxys
2. El dueño del website kerria seguir siendo capaz de saber ke ISP estas
   usando y donde vives, hasta ke estos tipos de proxys no sean publicos y
   solo puedan ser accedidos por los usuarios de ese ISP. Para la mayoria de
   los casos, hay una solucion - proxys publicos.
Puedes encontrar una lista de proxys publicos en cualkier sitio. Hay dos buenas
URLs por donde empezar:

1. http://www.theargon.com
2. http://www.cyberarmy.com/lists

Para configurar tu navegador web para usar un servidor proxy, encuentra tu
cuadro de dialogo en la configuracion (varia dependiendo del navegador)

Nota: algunos servidores proxy tambien soportan sesiones FTP (algunos
puede ke solo soporten FTP)

1. Si es dueño de un ISP y kiere empezar un proxy para este
2. Si kiere convertir su ordenador en un proxy publico

un solo ordenador. En ese caso, kerra convertir su ordenador en un servidor
proxy y decir a todos los ordenadores de la red ke lo usen como proxy. Asi
el resto de ordenadores de la red haran sus peticiones HTTP y FTP a traves

Puedes encontrar muchisimos Remailers Anonimos en http://www.theargon.com.

user: tu username
pass: tu contraseña

Tu recibiras una notificacion de Email desde anon.isp.ee cuando tu mensaje

Encriptacion?

Todo el mundo puede leer tu Email. Mientras es algun aprendiz kien te hackea

mundo ke usa PGP puede tener su propia clave PGP. Una clave consiste en

o simbolos.

Cookies?
========
Has pensado como esos websites en la red estan siendo "mas inteligentes" cada

sites recuerdan tu password por lo ke no tienes ke reescribirla cada vez,

compra virtual etc'.
Todo esto es debido a la cookies. Las cookies son pekeños archivos los cuales
un website puede pedirle a tu navegador ke los cree y recoger informacion de
ellas. Los websites pueden poner tu contraseña u otra informacion en esos
archivos.
Si tu no kieres tus co-workers (co-trabajadores) o ke otras personas cotilleen
y vean ke has visitado, ke articulos has estado comprando etc', debes
borrarlas cuando no las necesites.
En Unix, tus cookies normalmente se encuentran en algun lado de tu directorio
home (normalmente /home/tu-login/, /usr/tu-login o /usr/local/tu-login si
eres un usuario regular y /root si eres root, pero todos con acceso de
escritura a /etc/passwd puede cambiarlo).
En Windows y Mac, las cookies se encuentran en un sub-directorio del
directorio de tu navegador llamado cookies.

Nota 1: puedes decirle a tu navegador ke pregunte antes de aceptar una cookie.
Tan solo busca alrededor del menu de preferencias, lo encontraras
(hay muchos navegadores por ahi fuera asi ke no puedo dar una explicacion
detallada para cada uno).
Nota 2: si estas navegando desde un ordenador publico, no guardes ninguna
cookie, u otras personas seran capaces de cotillear y mirar en tus cookies
o incluso entrar en varios websites con tus contraseñas, numero de tarjeta
de credito etc'.

Un lector llamado Stone Cold Skunk me ha advertido ke el archivo cookies.txt
posiblemente se encuentre en el directorio netscape\users\default.
Esto ocurre cuando registras cuando te registras (Netscape te permite tener
multiples usuarios para el mism programa, cada uno con su propia canfiguracion etc')
sin dar un username.
El tambien me apunta ke algunos websites te rekeriran aceptar cookies
obligatoriamente para entrar.
Tambien, recomienda tener cuidado con el historial de tu navegador
(informacion de como borrarlo se puede encontrar en el capitulo "Donde Puedo

Apendice C: el modo +x
----------------------
En el IRC, es posible ponerte en modo +x escribiendo '/mode tunick +x'
(no incluyas las comillas y reemplaza tu tunick con tu nick propio.

en su lugar).

ver ke conexiones esta manteniendo tu ordenador. Una de ellas sera
la conexion DCC con ese otro tipo.

(Comunicacion directamente entre clientes -programas-), ke kiere decir ke
las acciones de DCC no se hace a traves del server, sino directamente
(piensa - por ke los dueños del server IRC kieren ke las personas

(1) Un poquitin de teoria, protocolos?

Para q tu y yo nos entendamos necesitamos hablar el mismo lenguaje espanyol o

necesitan hablar algun protolo, pero el mismo, algun ejemplo de protocolo

ultimo es el nuestro, el protocolo netbios, tecnicamente este protocolo va
encapsulado, o va dentro, o funciona, dentro de otro mas grande llamado TCP/IP.
(Mas info sobre TCP en el articulo de raizer TCP/IP basico).

Nota de imoen : el protocolo SMB es lo q se conoze como netbios sobre TCP/IP.

En concreto sera el protocolo llamado SMB (service manager block) el encargado
de dejarnos compartir archivos e impresoras y se implemeta sobre otros
protocolos como tcp/ip o netbeui, pero esto ya se lo dejo para la gente mas
elite.

La comparticion de ficheros y impresoras son servicios q corren en el puerto
(TCP) 139 que es conocido como el servicio de sesion NETbios. Este servicio
es usado para dar acceso local a los ficheros e impresoras compartidos en una
red de area local,  una red mas grande(wan), o incluso internet.

Que es SAMBA?
Bueno samba es un servidor(cliente?) para el protoclo SMB(o netbios sobre
tcp/ip).Sobre esto ya hemos hablado mucho en el ezine asi q n voy hablar
mas de ello todos sabeis q es lo q usa windows para compartir archivo e

entrar en los recursos compartidos desde linux jajajaja (y ya van 3
articulos sobre los recursos compartidos xDD)

Instalar SAMBA?
Hay varias formas y ademas depende de la distro de linux q tu tengas, lo
recomendable es q te descarges el fichero desde www.samba.org, el codigo
fuente y lo compiles tu, pero bueno a falta de eso tambien valen los binarios
y si aun lo quieres hacer mas facil pues un paquete rpm,(siempre y cuado tu
distro lo soporte).
Prueba a ver si tienes el samba instalado mediante find / -name samba.
Te recomiendo q antes de usar Samba tomes contacto con los comandos de
linux. A ver si para el proximo numero hablamos sobre esto, pero hay 100 mil
manuales y articulos sobre ello, y sobre samba no te voy a explicar nada
mas de lo que ponga en los How TO.
Pasos generales para instalar el samba si te decargas el tar.gz  q ocupa en
su version 2.2.1 unos 6 megas.
Desomprimelo en un directorio, mediante elcomando apropiado o desde la X
si lo prefieres (tar -zxvf nombre_del_fichero.tar.gz)
Como root los colocamos en el direcctorio origen del samba y tecleamos
./configure, despues con make crearemos los binarios.
Y por ultimo lo instalamos con make install. Proceso terminado.

Configurar SAMBA?
Para instalar samba hay q centrarse en dos cosas uno, el fichero de
configuracion smb.conf y la forma en que queremos q se inicie el servidor
samba(hay dos formas desde el inetd, o desde comandos, aqui una lista de
ordenes para controlar este servicio:

smb start para iniciar el sistema
smb stop para detenerlo, habra q detenerlo siempre q hagamos cambios en la
configuracion
smb status para ver el estado del servidor
smb restart para reiniciar el servidor samba

El fichero de confogiracion es un aricho de texto, y antes de hacer cambios
os aconsejo hacer una copia del original o del vuestro, con la siguiente
orden :  cp etc/smb.conf  etc/smb.conf.original

Vueno q de codsas tienen ehhh, ya veras q es bastante simple configurarlo,
el caso es que se puede sustitir a windows NT como servidor de dominio o
autentificacion de usuarios.

Vemos q hay tres apartados global, printers y shares, vamos a comentarlos,
aunque la mayoria de opciones por defecto q trae samba pueden valernos.

            subred pues entonces 172.16.0. fijate q termien en .

            User seguirdad del tipo usuario contrase¤a
            Share solo se espera una contrase¤a valida
            server se desvia el control de usuarios a otro servidor
            por ejemplo un windows NT o otra maquina linux.
Log file y max log file:  Sirven para definir el fichero de log y el tama¤o
            maximo de dicho fichero (/var/log/log.%m  o max log size =100)

Domain controller: Si en nuestra red disponemos de un servidor win NT q sea
            controlador de dominio aqui deberiamos de poner su nombre
Interfaces: pues para seleccionar nuestra tarjeta de red en caso de disponer
            varias.

[printers]
Desde aqui configuraremos las impresoras compartidas mejor q partiendo de lo
