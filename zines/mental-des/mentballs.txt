Bueno voy a empezar dedicandome un poco la instanlacion de Linux.(solo lo
basico).

Aca va un breve rese¤a tecnica y practica:

CONSEJOS: Antes de arriesgarse a perder toda la info de su HD asegurense en
que formato esta su particion, ya que Linux soporta particiones FAT16, y en
este caso solo tendriamos que utilizar el FIPS para reparticionar; si es en
FAT32 NO HAY OTRA SALIDA que empezar todo de cero.<------ Esto es por si
buscan un proyecto serio con Linux si no pueden montarlo en un directorio de
DOS para ir ambientandose.

Vamos a los bifes, la instalacion consta para mi, de tres puntos fundamentales
para cualquier novato. La primera es el reparticionamiento del HD, yo
recomiendo usar el FDISK ya que es un poco mas complicado pero lo veo mas
efectivo. Bueno al cargar FDISK nos va a aparecer algo asi:

#fdisk
Command (m for Help)
Se hace de esta manera siempre y cuando usemos /dev/hda como default si no 
tendriamos que hacerlo de esta manera: --> #fdisk /dev/sda

Ya adentro del FDISK tecleamos "m" y nos va a aparecer lo siguiente :

a      Toggle a bootable flag
b      Edit BSD disklabel
c      Toggle the dos compatibility
d      Delete a partition
l       List know partition types
m     Print this menu
n     Add a partition
p     Print the partition table
q     Quit without saving changes
t      Changes partition system id
u     Changes Display/entry units
v     Verify the partition table
w    Write table to disk and exit
x      Extra funcionality (experts only)

------------------------------------------------------------------------------

Ahora ya sabemos que comandos vamos a usar empecemos con las particiones:

Command (m for Help): p

Nos da las particiones que ya tenemos (es decir la particion de DOS FAT16).
Hay que prestar mucha atencion a los bloques en los que terminan las
particiones ya que no se tiene que solapear, es decir si /dev/hda1 termina en
el bloque 180, /dev/hda2 debe empezar een el 181 para no crear conflictos.

A CREAR!!!!

Command (m for Help): n <----crea una nueva particion

UHHH Sorpresa que pasa ahora tenemos 2 particiones Linux Native, tranqui, es 
asi no hay nada mal. Ahora que tenemos las tres particiones debemos cambiarle
el system id a la ultima, lo que se hace de la siguiente manera:

Command (m for Help): t

Listo!!! Ya le hicimos lugar a linux ahora revisa que este todo bien
ejecuntando "p" para listar las particiones ya terminadas y si todo salio bien
usa el comando "w" para grabarlas al disco y salir de FDISK.

ATENCION: Una ves echas las particiones, si es que van a instalar otros S.O. 
lo tendran que instalar antes que a linux ya que este guarda el LILO en el 
MASTER BOOT RECORD y al instalar otro S.O. lo cambia y se perderia todo (bahh

La segunda parte importante es la seleccion de paquetes para su instalacion,
les recomiendo que a no ser que sepan mucho no deseleccionen los predetermina
dos ya que pueden alterar el funcionamiento del S.O. en esta parte se pueden
seleccionar individualmente los paquetes de cada categoria, si tenes alguna
duda sobre a que se refiere cada paquete presiona F1 y sale un menu de ayuda.
Luego de terminar la seleccion es posible que el sistema detecte para el 
correscto funcionamiento de una de las categorias, asi que no lo dejen de 
lado. ya que si no lo hacen no tendran el correcto funcionamiento 

Ahora el ULTIMO punto: Junta todos los manuales de tu equipo, placa de video, 
mother, micro, modem, placa de red, monitor, teclado, mouse y mucha, mucha
paciencia, ya que llego la parte en la que comenzaras configurando, el mouse
luego el teclado, la zona horaria, la configuracion de red(not dialup), la 
impresora y por ultimo el monitor y la placa de video. Esta ultima es una de

No voy a detallar estos metodos ya que no me entere de mucha gente que

Saliendo del tema instalacion (aunque nunca se deja de lado 100%) otro punto

ACA VAN LOS COMANDOS QUE USE YO:

MAKE CONFIG: programa interactivo, bastante sencillo (tipo multiple choice)
al que hay que responder con Yes, No o Module.

MAKE MENU CONFIG: Programa que se maneja por menus, los componentes estan 
divididos por categorias, y se maneja de la misma manera que el primero: con
Yes, No o Module.

MAKE XCONFIG: Es una aplicacion X, los componentes estan en una lista con 
diferentes niveles de menus y los seleccionas con el mouse.

NOTA: Recomiendo el primero ya es el mas sencillo. :-)
