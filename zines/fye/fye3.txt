 [02]-----------------------<<<< Curso de C >>>>------------------------------
								     by ThEye

  Holas de nuevo!!! aka prosigue este pekeÒo curso de C el cual ya va en su
tercera entrega (como pasa el tiempo).En este capitulo abordaremos algunas
cosillas interesantes como son  getchar,putchar,el "super" goto,switch ,etc.
 Empezemos con getchar y putchar.
 Estas instrucciones son para el manejo de la entrada y salida de caracteres,
la forma en ke se manejan entrada y salida es muy simple...no importe donde
fue hecha la entrada o hacia donde se dirige la salida..siempre se tratan como
flujos de texto o streams.Un flujo de texto obviamente es una secuencia de
caracteres separados por lineas y en cada linea hay cero o mas caracteres
seguidos por un caracter NUEVA LINEA.
La biblioteca standar nos da como opciones mas simples para el manejo de E/S
las instrucciones getchar y putchar.Cuando se invoca a getchar este lee el
siguiente caracter y le asigna su valor de esta forma:

c=getchar()

es la variable c la ke contiene el sig. caracter de entrada.La funcion
putchar escribe el sig. caracter en la salida:

putchar(c)

esto escribe el contenido de la variable c en la salida.
Es importante indicar ke la entrada y la salida no son necesariamente siempre
el teclado y el monitor...esto puede ser variado.
Voy a poner un ejemplito de getchar en un programa...este ej. pertenece al
libro : Lenguaje de Programacion C por D.Ritchie y B.Kernighan.

#include <stdio.h>
main()
{
   long nc;              /* declaramos nc como un entero largo */
   nc=0;                 /* le damos el valor 0 a nc */
   while(getchar()!=EOF) /* mientras el caracter de la entrada no sea un EOF continua el ciclo (NOTA: el signo != significa "no igual a ") */
	++nc;            /* este es el ciclo...si no se recive un EOF nc se incrementa en uno */
   printf("Los caracteres fueron: %ld\n",nc);   /* en algun momento el ciclo llega a su final por lo tanto el programa cumple su funcion y nos informa cuantos caracteres entramos */
}

  Ok...lo ke hace este programa,como pueden ver,es contar los caracteres ke se
le dieron a la entrada,y almacena ese valor en la variable nc hasta ke el
usuario envia un caracter de fin de archivo (EOF),esto kiere decir,hasta ke el
usuario presione CTRL-Z (DOS) o CTRL-C (Linux).

  Ahora veamos un ejemplo donde aparezcan getchar y putchar (mismo libro):

#include <stdio.h>
main()
{
    int c;
    while((c=getchar())!=EOF)
	putchar(c);
}

  Este ejemplo es casi lo mismo ke el anterior solo ke aka hacemos ke putchar
repita el caracterer ke llego a la entrada,en la salida hasta ke c sea igual
a EOF.
  Kiero aclarar ke en este ejemplo se declaro a c como un valor lo
suficientemente grande como para contener cualkier valor ke getchar le asigna,
por eso se declaro int y no char ya ke char no podria haber contenido a EOF
ademas de culkier otro caracter.EOF es un entero definido en stdio.h pero el
valor numerico no importa mientras no sea el mismo ke algun valor tipo char.
Como se usa la constante simbolica el programador se asegura ke nada depende
del valor numerico.
  En C cualkier asignacion como:

  c=getchar()

tiene un valor y es el del lado izquierdo despues de la asignacion.Eso kiere
decir ke una asignacion puede ser parte de una expresion mas larga como la
ke utilizamos en el programa:

  while((c=getchar())!=EOF)

como se pueden dar cuenta,c=getchar() esta encerrado entre () debido a ke la
precedencia de != sobre = es mayor,ke significa esto?? significa ke si
hubieramos hecho esto:

  while(c=getchar()!=EOF)

primero se hubiera probado la asignacion : getchar()!=EOF y luego:
c=(getchar()!=EOF)...esto hubiera causado ke el valor de c fuera 0 o 1 , eso
depende de si encontraba o no un caracter fin de archivo.
  Bueno con getchar y putchar solamente, se puede escribir una gran cantidad
de codigo bastante interesante...por ej. algo chico:Se puede hacer un
programa ke cuente el numero de lineas,espacios en blanco,tabuladores,etc. ya
ke al hacer verificaciones de las variables les preguntamos si contienen estos
caracteres...como es eso?? bastante simple...si tenemos un getchar ke recive
un caracter y keremos saber si ese caracter es una nueva linea ,usamos
para eso los secuencias de escape de printf ke ya se indicaron en el anterior
numero de FYE...veamos como seria:

 c=getchar();
 if(c=='\n')
    proposicion del if...

  Un caracter escrito entre '' representa un valor entero igual al valor
numerico del caracter.A esto se le llama constante de caracter...Veamos por
ej.:'A' es una constante de caracter,ke en ASCII ekivale a 65...obviamente es
mucho mejor escribir A ke 65.Lo mismo ocurre con \n ke es el caracter de
nueva linea cuyo valor en ASCII es 10.
  Veamos ahora algo de los arreglos,con un ejemplo donde se ve claramente
ke es lo ke hacen estas cosas (ejemplo sakado del libro de siempre):

#include <stdio.h>
main()
{
    int c,i,nwhite,nother;
    int ndigit[10];

    nwhite=nother=0;
    for(i=0;i<10;++i)
	ndigit[i]=0;
    while((c=getchar())!=EOF)
	if(c>='0' && c<'9')
	    ++ndigit[c-'0'];
	else if(c==' '||c=='\n'||c=='\t')
	    ++nwhite;
	else
	    ++nother;

    printf("digitos =");
    for(i=0;i<10;++i)
	printf("%d",ndigit[i]);
    printf(",espacios en blanco= %d, otros= %d\n",nwhite,nother);
}

  Lo que hace este programa es contar digitos(numeros),espacios blancos y
  otros.
  int ndigit[10] declara a ndigit como un arreglo de 10 enteros.Los subindices
de arreglos empiezan desde 0 por lo tanto son: ndigit[0],ndigit[1]...y asi.Un
subindice puede ser cualkier expresion entera como i por ejemplo.
  if(c>='0' && c<='9') determina si c es un digito o no...si lo es ndigit toma
el valor numerico c-'0'.Ver si un caracter es nidigo,blanco u otra cosa se
hace con:

  if(c>='0' && c<='9')  /* NOTA: && significa Y */
	++ndigit[c-'0'];
  else if(c==' '||c=='\n'||c=='\t')
	++nwhite;
  else
	++nother;

  Ahora vamos a ver la proposicion switch.
  Esta proposicion es una decision multiple ke decide ke parte del codigo
debe ejecutar (ke me perdone la gente ke sabe mucho mas de C por esta
definicion tan simple...pero bueno,es acertada o no?).
  Su forma es la siguiente:

  switch(expresion){
	case exp-const: proposiciones
	case exp-const: proposiciones
	default: proposiciones
  }

  Cada CASE se etiketa con uno o mas valores enteros.Si un case coincide con
la expresion la ejecucion comienza alli.Todo case debe ser diferente a los
demas.Default se ejecuta si ninguno de los otros satisface (es optativo,si
ningun case satisface y no exite default no pasa absolutamente nada) y ademas
puede estar en cualkier lugar...no necesariamente al final.
  Veamos ahora un programa ke nos muestre la utilizacion de switch:

#include <stdio.h>
main(){
int c;
printf("\nEs ThEye el mas bakan de todos??(s,n)\n");
c=getchar();
switch(c){
case 's':
printf("Totalmente de acuerdo contigo!!!\n");
break;
case 'n':
printf("Fuck You!!!\n");
break;
default:
printf("\nTan dificil es para ti responder con s o n ???\n");
break;
}
}

  No hay necesidad de explicaciones...el uso de switch en este programa es
bastante claro...despues ke se imprime la pregunta Es ThEye el mas bakan de
todos?,se hace uso de getchar para capturar el proximo caracter ke llegue a la
entrada,luego switch verifica cual caracter es el ke se eligio y si coincide
con las alternativas dadas (s y n) se muestran unos mensajes relativos a la
respuesta,si se elige cualkier otro caracter se obtiene un mensaje ke nos
indica ke debiamos solo elegir s o n.
  Ya ke los case son solo etiketas,despues de la ejecucion de una de ellas
el proceso sigue con la siguiente,para evitar esto se hace uso de BREAK ke
termina con el switch...para esto mismo tb se puede hacer uso del return.
Break no es una instruccion propia de switch si no ke es independiente y tb
se le puede dar uso para terminar con ciclos while,for y do.Es bueno hacer
notar ke tb. hize uso de un break al final de default...esto al parecer es
ilogico...pero no lo es.Agregar un break al final del default es una practica
ke se recomienda ya ke en algun momento,tal vez,se decida ampliar el codigo
agregando nuevos case despues de default por lo tanto el agregar tb. el break
en default nos previene de posibles errores.
  Podemos ver un uso del switch semejante a este en el zap ke CrackX ha
escrito para su articulo de Borrado de Huellas Basico en Linux.
  Mmmm...veamos ahora la archiconocida instruccion GOTO :P .Siempre se dice
que goto es una instruccion infinitamente abusable ya que siempre hay lugar
para ponerla,pero en la practica nunca es bueno hacerlo ya que solo enreda
el codigo y no lo hace tan facil de entender,asi que ya saben:evitar hacer
codigos "spaghetti" (osea con muchos goto) ya que siempre se podra hacer el
mismo codigo sin el.
  Veamos un peque~o ejemplo:

#include <stdio.h>
main()
{
    int c;
    pregunta:
    printf("FYE rulez ?\n");
    while((c=getchar())!=EOF)
	if(c=='s')
	    goto yea;
	else 
	    goto fuck;
	
    yea:
	printf("Yea Yea :D\n");
	goto fin;
    fuck:
	printf(">:| ,solo puedes responder afirmativamente xDDD\n");
	goto pregunta;
    fin:
}

  Ahi esta el ejemplo,como ven el goto lo unico que hace es enredar el codigo
asi que si hubieramos precindido de el hubieramos obtenido un codigo mas
simple y legible.
  Bueno,aca TERMINABA el articulo pero he revisado la extension de este y lo
encontre algo corto,asi que vamos a analizar dos peque~as cosillas mas :).
  Un programa no tiene gracia si el usuario no puede ingresar lo que desea que
el programa realize,ya hemos revisado getchar,switch pero existen mas
instrucciones que nos permiten hacer interactuar a programa y usuario.Veamos
una de ellas,la funcion scanf.
  La funcion es una entrada con formato (no tengo que explicar esto no?) y
a la vez entrada analoga de printf.Scanf lee caracteres desde la entrada 
estandar,los interpreta y los guarda en una variable (la forma en que
usaremos scanf es muy similar a la del input de basic),veamos un ejemplo:

#include <stdio.h>
main()
{
	int n;
	printf("Give me a number please\n");
	scanf("%d",&n);
	if(n<0)
		printf("Your number is negative\n");
	else if(n==0)
		printf("Your number is zero\n");    
	else
		printf("Your number is positive\n");
}

  Como pueden ver,aca el programa pide que se ingrese un numero,el numero
ingresado lo obtiene scanf y lo almacena en la variable n que es un
entero,luego este es analizado en un if y dependiendo de si es negativo,cero o
positivo recivimos la respuesta correspondiente.
  Hay muchas otras formas de usar scanf,pero al final su objetivo es el mismo
asi que no las explicare porque para ser sincero ni siquiera yo se usarlo muy
bien xDDD , :P .
  Hablemos ahora de argc y argv.Estas dos variables de declaran dentro de
la declaracion de parametros de la funcion main y lo que hacen es recivir los
parametros que ingrese el usuario junto al programa ejecutable.
  Argc recive como parametro el propio nombre del programa por lo tanto su 
valor es siempre mayor o igual a 1,argv por su parte es el resto de los 
parametros ingresados por el usuario.Watchemos un ejemplo xDDD :

#include <stdio.h>
#include <stdlib.h>
int main(int argc,char *argv[])
{
	if(argc!=2){
	   printf("Debias ingresar tu nombre!!!\n");
	goto fin;
		}               
	printf("Fuck You %s !!!!!!!!!\n",argv[1]);
fin:
}

  Como podemos ver,aca declaramos otra libreria:stdlib.h lo que hace
esta libreria es declarar funciones para asignacion numerica,asignacion
de memoria y cosas parecidas.La necesitamos para esto:

int main(int argc,char *argv[])

esto nos permite definir a main como un int y hacer uso de las caracteristicas
de las funciones en C,es decir tener argumentos y devolver valores.La linea
del if cuenta los parametros ingresados por el usuario,ya que sabemos que
el nombre del programa es uno de ellos ,nosotros queremos que nos ingrese un
nombre por lo tanto el numero total de parametros debe ser 2.Como sabemos en
C se empieza a contar desde 0 ,por lo tanto el 0 le corresponde al nombre
del programa y 1 (que lo leemos en la linea del segundo printf) es el segundo
parametro osea el nombre que ingresa el usuario.

 [04]-------------------------<<<< Tarreando >>>>-----------------------------

			      [][] TARREANDO [][] 
				   by NDR113            
 
A continuacion presentare para todos ustedes esta nueva seccion, un intento 
por introducir en el armado de computadoras personales para quienes este  
interesados en el hardware, en personalizar su propia maquina y por supuesto 
dejar de gastar plata de mas por una PC de marca, maquinas que no siempre 
tienen los mejores componentes y que son en su mayoria bastante poco 
escalables o actualizables. 
 
1.0 CPU 
Central Unit Process, o dejemoslo mejor en simplemente el procesador. Este es 
en si el "computador". Este chip esta contruido sobre una oblea de silicio  
combinada con otros materiales para alterar sus capacidades electricas, a 
la que se le graban los conductos electronicos y sus microtransitores mediante 
un proceso de fotolitografia, trabajando el chip a traves del dise~o de
tuberias   de sus microtransistores, como quedo explicado de forma elemental
en el articulo  "Fundamentos e Historia de la Informatica" en FYE 1. 
El procesador consta de una unidad logica y una aritmetica, 
y antiguamente se le acompa~aba de una co-procesador matematico especifico 
o FPU (Float Point Unit, o Unidad de Punto Flotante, pues era encargado del 
manejo de los decimales). Desde la aparicion del Pentium el FPU y el CPU 
fueron integrados. 
Actualmente el mercado de procesadores esta basado basicamente en la  
arquitectura x86, la que es liderada por Intel Corp. y su linea 
Pentium (...Celeron/II/III/Xeon...). Otros fabricantes que comparten el
mercado  son AMD y en menor medida Cyrix. 
La velocidad de los procesadores esta dada por Mhz o millones de herzios (o 
frecuencias) por segundo, y aun cuando esta es generalmente relacionada con  
el poder del tarro, este numero es solo uno de los factores a considerar, pues  
la relacion entre los Mhz de distintos procesadores no es es siempre una  
referente valida para conocer su verdadero poder o rendimiento. 
Debe considerarse tambien el tama~o del cache interno del procesador, el que  
cumple la funcion de antememoria primaria entre los demas componentes y la 
circuiteria del procesador mismo; tambien es importante evitar los cuellos 
de botella en cuanto a la velocidad a la que se comunica el CPU con los  
perifericos y el bus en general, puesto que el numero de Mhz que se muestra 
es la velocidad interna, mientras que la velocidad de Input/Output es
generalmente  mucho mas baja. Es asi que la mayoria de los CPU mantiene el
vinculo externo  con el bus a 66, 100 o 133 Mhz.  Ojo aqui: la relacion entre
Mhz del bus  y Mhz internos tiene cierta repercusion en el rendimiento, me
explico,  un procesador cuya velocidad de bus sea multiplo exacto de la
velocidad interna  gana en rendimiento. Por ejemplo: 
	 
	V. Interna      V. Bus  Multiplicador 
	100 Mhz         66 Mhz  1.5 
	133 Mhz         66 Mhz  2.0 
	166 Mhz         66 Mhz  2.5 
 
Mientras que el primer y ultimo CPU multiplican por decimales, el de 133 lo
hace  por un entero, lo que le da ventaja en rendimiento. Es por eso que la
diferencia  en rendimiento entre un P100 y un P133 es mayor que entre el P133
y el P166.  Tambien la cantidad de bits que puede procesador un CPU es
indicador de su poder,  actualmente en los CPU Pentium y AMD K6 en adelante
son 64 bits por ciclo -herzio-.  En contraste con los 386/486 de 32 bit o los
286 de 16 bits.    
1.1 Que Comprar? 
Sea el CPU que compres en dos a~os ya estara pasado.. por que? Muchos
factores,  pero quizas el resumen de todos estos esta contenida en la conocida
Ley de Moore,  la que predice que cada 18 meses la capacidad de los
procesadores se duplica,  al duplicarse la la cantidad de transistores que
contiene los mismo.   Entonces, piensa bien y equilibra Precio/Rendimiento.
Generalmente lo ideal  es comprar el penultimo o antepenultimo CPU que se haya
lanzado, lo que que   en terminos de plata no es muy economico, pero
seguramente te durara.  Actualmente un PIII500 es una excelente opcion, pero
con el advenimiento de  los procesadores de 1 Ghz (1000 Mhz) por parte de IBM,
AMD e Intel, quizas  esperar por uno 700 Mhz (AMD o Intel) seria bueno. Que no
recomiendo? Nada bajo los   400 Mhz que no sea PII/PIII o AMD K6-II/Atlhon.
Celeron NO! ya que este procesador   para mi gusto es bastante lento en
comparacion con otros dise~os a la misma   velocidad, asi que no te impresione
un Celeron 400 o algo asi, ya que su   dise~o de bajo costo es una version
reducida del PII y no es suficientemente   eficiente como este ultimo a la
misma velocidad.   
1.2 Procesadores Multimedias 
3D Now? SSE? MMX? Sobre la ya betusta arquitectura x86 (que data de los '70) 
se han a~adido nuevas instrucciones que ayudan al procesamiento paralelo 
y aceleran la paginacion de ordenes, lo que majora en gran parte el
rendimiento  especialmente en las aplicaciones multimedias. Comenzo con la
aparicion de MMX  en los Pentiums y en las subsecuentes generaciones de Intel.
Luego AMD a~adio  sobre MMX en sus procesadores 3D Now! para facilitar la
generacion de graficos  tridimensionales. Finalmente el PIII incluye SSE, una
serie de nuevas intrucciones  dise~adas para mejorar el manejo de graficos,
pero que en terminos de rendimiento  real no mejora de forma visible la
velocidad en aplicaciones no-dise~adas   especialmente para SSE. 
Como se aprovechan estas nuevas intrucciones? A traves de codigo dise~ado y 
escrito para ellas -menos compatible- o a traves de algun API (Aplication 
P Interfaz) que maneje el codigo normal y lo aplique a las capacidades de tu 
procesador, ejemplo: DirectX para juegos bajo Windows. 
 
 
2.0 Motherboard (Placa Madre) 
Aqui va tu procesador, memoria, controladores de disco... todo. Ojo en esto:
para  recortar costos, muchos armadores y fabricantes de PCs estan integrando 
componentes como modems, tarjetas de sonido, y tarjetas de video en la misma 
placa, lo que aun cuando hace el PC mas barato, tambien lo hace mas lento, 
menos expandible y menos compatible, por cuanto el trabajo de esos dispotivos 
se limita a unos cuantos conjuntos de chips en la placamadre en vez de contar 
con todo lo que trae el dispotivo real en su tarjeta dedicada; esto le resta 
poder al procesador asi como memoria RAM (dispositivos de video en la placa 
madre no incluyen memoria propia, asi que toman MB de la RAM del sistema. 
Por ejemplo el ThEye, tiene 32 MB EDORAM pero puede utilzar 30 por que los
otros  2 los toma el sistema de graficos SiS que tiene en su placa madre <-
XDD  )  Asi que, deja claramente claro que quieres tarjetas reales, no
chipsets   integrados 
Luego comienza por buscar una placa que soporte la velocidad y tipo de
procesador  que piensas utilizar, pero que tambien tenga cierta capacidad de
actualizacion  para en el futuro solo tener que cambiar el CPU y no tb la
placa madre.  Asi que asegurate que soporte mayores Mhz que los del CPU que
instalas.  Atencion si compras un procesador Atlhon ve que sea abiertamente
compatible  con el procesador de AMD, pues utilizan otros chipsets y una
arquitectura  de bus mas rapida que las placas normales para procesadores
Intel.  Tambien debe contar con un buen bus, es decir, tener la expansibilidad
en  cuanto a los slots PCI e ISA que necesitas, asi como tb un bus AGP y  
capacidad para USB. 
 
2.1 ISA, PCI, AGP, USB, etc 
ISA (Industrial Standar Arquitecture): Bus de 8 o 16 bits (la extension EISA 
soporta placas de 32 bits). Es utilizado primariamente por dispositivos 
antiguos, como tarjetas de sonidos, modems o NICs (Tarjetas de Interfaz de
Red).  Fisicamente lo reconoces por ser el conector mas largo, ancho y
generalmente  de color negro. 
 
PCI (Personal Computer Interfaz): Bus de 16 y 32 bits, utilizado por
dispositivos  mas modernos debido a su mayor rapidez y ancho de banda
comparado con ISA.  Aqui veras las nuevas tarjetas de sonido, soft/win modems,
y tarjetas de video  relativamente antiguas. 
 
AGP (Advanced Graphics Port): Bus para tarjetas de video y aceleradores
graficos,  que por su amplio ancho de banda, velocidad (133 Mhz) y por contar
con un dise~o  mas directo hacia el procesador, se ha vuelto el estandar en
las tarjetas de 3D  de hoy en dia... un "must-have". 
 
USB (Universal Serial Bus): Con una ancho de banda de 12 Mbit/seg el USB a
tendido  a relegar los antiguos conectores como el puerto paralelo y serial
para aquellas  tareas que necesiten de velocidad, como Scanners, Impresoras de
alta resolucion,  monitores digitales, camaras digitales, dispositivos de
almacenamientos, etc.  Su dise~o se basa en una especie de hub o margarita, es
decir a cada uno de los   dispositivos que se conectan se le puede conectar
otro , y asi a su ves otro,   etc. 
 
COM, LPT, AT, PS/2: Puertos externos de baja velocidad de orden basico. 
	 
	     
Serial          Mouse Seriales, Joystick, Modems Externos, Cables Null         
		Conectores: DB-15 
		Conocido como: COM - ttyS 
 
Paralelo        Impresoras, Scanners, Cables Null 
		Interfaz: RS-232 (25 pines) / RS-449 (37 pines) 
		Contectores: DB-25 / DB-37 (segun pines) 
		Conocido como: LPT 
 
AT              Teclados (En desuso actualmente en favor de PS/2)          
		Conectores: DB-5 
		Conocido como: Din 
 
PS/2            Teclados, Mouse 
		Conector: PS/2 de 6 pines 
 
*** El numero que acompa~a a DB es la cantidad de pines del conector. 
 
Quizas la unica variante en estos puertos son los estadares ECP/EEP,
utilizados  para darle capacidad bidireccional a los puertos LPT para mayor
rendimiento  y capacidad en las impresoras actuales y scanners. 
 
3.0 Memoria RAM 
La RAM, o Random Access Memory, es la memoria que recive las instrucciones 
y procesos del cpu para su almacenamiento momentaneo. Su velocidad 
y capacidad se traduce en claras mejorias de rendimiento. 
Aqui la cantidad es tan importante como la velocidad, pues los chips de ram 
van en un amplio rango de velocidades, desde la antigua EDO  de 70
nanosegundos a los chips de SDRAM actual, que tiene velocidades de respuesta
de menos de 10 ns. Cantidad? Actualmente no es recomendable nada por debajo
los 64 MB   de SDRAM. Puedes intalarla en SIMMS (72 pines) o en DIMMS (168
pines)   
4.0 Almacenamiento Fijo  
Hablamos de disco duro, el medio de almacenamiento magnetico masivo mas 
utilizado y economico. Aquella cosa que nunca es tan grande como queremos 
que sea, y sin duda es lo primero que se agota en la vida de un pc. 
En cuanto a espacio solo hay una regla: Compra el disco mas grande que tu 
bolsillo pueda sustentar!. En cuanto al tipo de disco, o mas bien su
controlador  IDE o SCSI, podras ver que los IDE son mucho mas economicos, pero
los SCSI  son mas rapidos y eficientes debido a su topologia y ancho de banda
(hasta 40  Mb/seg, en comparacion con 2 Mb/seg de los controladores IDE). 
Otro factor que tiene relacion con la velocida del disco son los RPM 
o Revoluciones por Minuto, es decir velocidad de rotacion de los platos 
que componen los discos. Actualemente los discos giran desde 5.000 RPM hacia 
adelante, dependiendo del modelo.

  En este articulo pretendo explicar de una manera simple el modelo ISO/OSI de
comunicaciones entre computadores.Empezemos con unas aclaraciones...
  La ISO (International Standars Organization) es como su nombre lo dice el
Organismo Internacional de estandares globales para comunicaciones e
intercambio de informacion.ANSI es el miembro de USA en la ISO.El modelo
ISO/OSI (International Organization for Standarization's Open Systems
Interconnection) es uno de los modelos propuestos por la ISO mas ampliamente
aceptados.Veamos un eskema de este:

		      ----------------------------------
		     |        Capa de Aplicacion        |
		     |              Capa 7              |
		     |----------------------------------|
		     |       Capa de Presentacion       |
		     |              Capa 6              |

		     |              Capa 5              |

		     |        Capa de Transporte        |
		     |              Capa 4              |  
		     |----------------------------------|
		     |           Capa de Red            |
		     |              Capa 3              |  
		     |----------------------------------|
		     |      Capa de Enlace de Datos     |
		     |              Capa 2              |  
		     |----------------------------------|
		     |           Capa Fisica            |   
		     |              Capa 1              |
		      ----------------------------------

  Bueno por su nombre tal vez no dicen mucho...asi ke veamos ke hace cada
una.

1) Capa Fisica: mecanismos para comunicarse con el medio de transmision y la
interfaz de hardware.Define procedimientos fisicos,electricos,mecanicos y
funcionales ke se usan para conectar el ekipo. Osea el medio fisico por donde
viaja la informacion.

2) Capa de Enlace de Datos: valida la integridad del flujo de los datos de un
nodo a otro.Esta capa se divide en dos,pero para ke verlos??????...por si les
interesa,aca actuan los switches :D .

3) Capa de Red: define los protocolos de enrutamiento de datos ke aseguran ke 
la informacion llegue al nodo destino correcto y maneja los errores ke surgen
durante la comunicacion.Aca actuan los routers :) .

4) Capa de Transporte: aka van los protocolos para la estructurazion de los
mensajes y verifica algunos errores logrando asi supervizar la validez de la
transmision.

5) Capa de Sesion: coordina las comunicaciones y mantiene la sesion el tiempo
ke sea necesario.Ejecuta funciones de seguridad,ingreso de usuarios y tareas
administrativas.

6) Capa de Presentacion: determina la manera de como los datos son formateados,
presentados,convertidos y codificados.

7) Capa de Aplicacion: define como la red interactua con la aplicacion,aka
se incluyen correo electronico,programas emuladores de terminal y otros.Esta
capa no interactua con las demas capas si no ke tal como esta dicho lo hace
con los programas del usuario.

  La numeracion de las capas puedes encontrarla a la inversa,bueno eso no 
importa mucho pero lo digo para ke luego no se enreden.

  Pues bien,ya conocemos el modelo ISO/OSI y la funcion de cada capa...pero 
en la practica ke pasa???...todos sabemos ke internet esta basado en el 
mundialmente conocido protocolo TCP/IP...ya y esto ke tiene ke ver con el
ISO/OSI???...mucho pues...este protocolo trabaja dentro de este modelo.Por ej.
IP trabaja en la capa de red (la capa ke define los protocolos de enrutamiento
de datos) y TCP en la capa de de transporte (la capa donde van los protocolos 
para la estructurazion de los mensajes y ke se preocupa de supervizar la 
correcta transmision de datos) asi ke es el protocolo "seguro" del TCP/IP
ya ke como todos sabemos,IP no nos da ninguna garantia de ke nuestros datos
lleguen a destino sin embargo TCP si ya ke esta orientado a la conexion.
  TCP/IP actualmente es tan importante ya ke permite ke redes autonomas se
puedan conectar a internet o entre si para formar intranets privadas.Hablando
de una forma simple,TCP/IP oculta los routers y las arquitecturas de redes ke
hay detras de estos dando la impresion de una sola gran red.TCP/IP tb. tiene
un modelo como ISO/OSI...veamos una comparacion:

------------------------------------------------------------------------------      
      ISO/OSI                                  TCP/IP
------------------------------------------------------------------------------

		       |           |                              |

  uta ke dibujo feo xDDDDD...pero bueno esta custion no es una revista de arte
asi ke da lo mismo...yap veamos algunas cosillas del modelo TCP/IP...

1) Nivel Fisico: El medio fisico por donde viaja la info. osea el cableado.
2) Nivel de Vinculo de Datos: Aka estan las tarjetas de interfaz de red.Hay
ke saber ke RARP significa Protocolo de Resolucion de Direcciones en Reversa,
o sea convierte direcciones fisicas de red en direcciones IP y ARP es el
Protocolo de Resolucion de Direcciones ke convierte direcciones IP de 32 bits
en direcciones fisicas de red (por ej. direcciones de Ethernet de 48 bits).
3) Nivel de Red es donde va IP el kual enruta los datos hacia donde deben

4) Nivel de Transporte: Como se ve aca van los protocolos para estructurar los

a los programas de aplicacion de red,y por supuesto ambos dependen de IP.
5) Niveles de Aplicacion: Aca estan las Aplicaciones de Red y es donde actuan
los niveles de Sesion,Presentacion y Aplicacion del modelo ISO/OSI.

1.0 Que es Linux?
Linux es un sistema operativo multiusuario y multitarea, cuyas bases estan en
el legendario UNIX, con el que comparte grandes similutes, ya que quien
originalmente escribio linux, el noruego Linus Tordvals, penso en crear un OS
podero y liviando similiar a UNIX que pudiera correr en su computadora
personal de mediados de los 80s. Linux se basa en un kernel que es la unidad
principal y central de los procesos, un shell (similiar al interprete de
comandos de DOS/Windows) y una interfaz que puede ser directamente la consola
en modo texto o mediante el uso de un ambiente grafico como X Windows y las
interfazes como Gnome o KDE que corren sobre el, facilitando el uso del
sistema. En cuanto a los comandos, Linux guarda cierta lejana semejansa con
MsDOS, el que se baso en QDos, que a su ves de vaso en CP/M, este ultimo el
padre de Unix, y por lo tanto abuelo de Linux....
Linux nacio como un software de codigo abierto y libre distribucion, asi que
es en resumen gratuito y su codigo puede ser modificado de forma libre y sin
restricciones, lo que le ha dado un impulso muy fuerte en la comunidad de
desarrolladores y programadores, por lo que ha crecido de manera muy fuerte
las aplicaciones y versiones de linux.
Puedes encontrar paquetes o distribuciones que tratan de reunir y organizar
un conjunto de aplicaciones para armonizar y hacer mas facil el uso e
instalacion del OS. Distribuciones populares son: RedHat, Caldera Open Linux,
TurboLinux, Mandrake, SuSe y Debian, entre otras. Ahora revisemos lo basico.


2.0 Conceptos Fundamentales
Por ser un OS multiusuario, linux establece distintas cuentas de acceso a la
computadora, con lo que cada usuario tiene sus propias preferencias y permisos.
El usuario o cuenta que maneja todo el sistema sin restriccion root, conocido
como "super usuario".
Para identificarte ante el sistema debes logear, es decir, dar tu nombre de
usuario y password correspondiente para entrar al sistema. Como el sistema en
este caso es propio, obviamente la cuenta root sera la predeterminada para tu
uso, con lo que tu control sobre el tarro y sobre otros usuarios es total.
Aqui es donde recide la importancia de la cuenta root.
Una ves logeado o entraras a una interfaz grafica como KDE, Gnome o
WindowsMaker, o iras directamente a la consola, dependiendo de que tipo de
paquete de linux hayas instalado. Si te ha pasado lo primero, abre una ventana
de terminal para comenzar a moverte por los comandos fundamentales.
Antes que todo, hechemos una ojeada a la estructura del sistema de archivos
de linux. Aqui a diferencia de Windows, los directorios no estan agrupados por
programa, si no, estan organizados por tipo de archivo y funcion del mismo.
Nota: Linux es sensible a mayusculas, por lo que /dev no es lo mismo que /DEV.

3.0 Breve Descripcion de la Estructura

/bin    Aqui estan los binarios o ejectubles del interprete de comandos, los
	comandos de orden basico para el funcionamiento del sistema. Siendo
	extremadamente didactico podria decirse que es el equivalente del
	c:\windows\command del mundo Windows


/dev    Archivos de dispositivos. Estos en realidad son referencias a los
	recursos del sistema, como unidades de disco, modems, memoria,
	etc. El sistema lee la  informacion proveniente de estos "archivos" de
	dispostivos, de manera similar  a como lee datos en un archivo
	ordinario. Algunos ejemplos:
		
	/dev/hd[a,b,c..][0,1,2...]      Disco Duros (IDE)
					Las letras hacen referencia a
					si el disco es primario, secundario,
					etc; mientras que los numeros
					representan las particiones del disco.
					Si no hay numero, representa a todo
					el disco.
					P,ejemplo: la particion primaria de
					un disco duro IDE maestro podria ser
					/dev/hda1/
					Nota: Los discos SCSI estan bajo el
					prefijo sd, p.ejemplo: /dev/sda1

	/dev/fd[0-1]                    Discos Flexibles
					Aqui hay soporte para dos disqueteras.
					Ademas encontras muchos otros dispositivos
					que comienzan con fd, estos son representaciones
					de tipos especificos de discos flexibles,
					fd1H1440        Drive 1, Alta Densidad
					fd2H720         Drive 2, Doble Densidad, etc
	
	/dev/ttyS[0-9]                  Puertos seriales, conocidos en Windows o DOS
					como los COM. COM1 = ttyS0, COM2 = ttyS1 , etc

	/dev/lp[0-2]                    Puerto paralelo para impresion.
					Equivalente: LPT1 = lp0

	/dev/mouse                      

	/dev/console                    Esta es la referencia al monitor que esta
					conectado al tarro.

	/dev/null                       Pozo sin fondo de Linux, cualquier cosa
					que envies aqui se perdera. Podria decirse
					que es todo un agujero negro.

					
/etc    Configuraciones y archivos miscelaneos escenciales que tiene que ver
	con la  personalizacion del sistema, asi como secuencias de iniciacion,
	definiciones    de usuario y grupos, etc. P. Ejemplo:
	
	/etc/passwd     Informacion de los usuarios, el grupo al que pertenecen y su
			contrasena.

	/etc/group      Grupos de usuarios y sus permisos

	/etc/inittab    Configuracion del programa init, responsable de tomar el control
			una vez iniciado el sistema y terminado el montaje de discos,
			iniciando los scripts, y es responsabilizandose del manejo
			de los logins asi como del control de la secuencia de inicio
			y apagado.
			Analizaremos este archivo quizas mas adelante, ya que es bastante
			complejo y contiene informacion importantisima con respecto a
			la iniciacion de multiples consolas para multiples logings
			tanto a traves del teclado, como a traves de otros dispositivos
			como puertos seriales.
	
	/etc/fstab      Aqui estala informacion acerca de los sistemas de archivos
			y su lugar en el montaje. Informacion detallada en la seccion
			dedicada al montado de discos, mas adelante.

	/etc/lilo.conf  Informacion acerca del manejador de booteo LILO. Utilidad
			que permite iniciar multiples sitemas operativos en
			un mismo disco, manejando las particiones. Acerca de 
			LILO una descripcion mas precisa adelante.
			
				
/home   Aqui se mantienen por lo general los archivos y documentos propios de
       cada usuario por ejemplo, el usuario andres tiene su carpeta
       /home/andres para escribir sus documentos y demases.


/lib    Aqui estan guardadas las bibliotecas de funciones que los programas llaman al
	ejecutarse. Algo similiar -simplistamente hablando- a las librerias de Windows
	que se encuentran en c:\windows\system (extension .dll)


/proc   En este directorio se pueden encontrar los procesos que se ejecutan actualmente,
	asi como varibales del entorno que tienen que ver con el procesador y la
	memoria. Veremos en detalle los procesos y su administrador en Control de Procesos.
	

/tmp    Archivos temporales     


/usr    Aqui es donde reciden las aplicaciones y los archivos que los acompanan.

	/usr/bin        Los binarios (ejecutables) de las aplicaciones van aqui. Por
			lo que este directorio es donde se almacena el sofware.

	/usr/lib        Biliotecas equivalentes a las encontradas en /lib, y en
			efecto, cuando un programa necesita el codigo de la libreria
			la peticion se redirecciona desde /usr/lib hacia /lib.

	/usr/etc        Similar a /etc/ pero con la diferencia que los archivos aqui
			guardados no son configuraciones escensiales para el sistema.

	/usr/include    Contiene los archivos include necesarios para la compilacion
			de codigo C.

	/usr/man        Manual del sistema

	/usr/X11R6      Ficheros del sistema X Windows, el ambiente grafico de linux
			sobre el cual se utilizan las demas interfaces como Gnome,
			KDE, WindowsMaker, etc.Aqui encontrar los binario, archivos
			de configuracion y todo lo necesario para la ejecucion de
			X Windows, por lo que este directorio tambien tiene su propia
			estructura con /bin /include /man /share ... etc

	/usr/share      Organizado por subdirectorios, este contiene otros archivos
			utilizados generalmente por las aplicaciones de /usr/bin

	/usr/doc        Documentacion varia acerca de las funciones de las aplicaciones
			y sistema. Por lo general mayoria se encuentra en formato .HTML

/var                    Variables y temporales. Tambien aqui reciden los procesos de
			spooling como la colilla de impresion en /var/spool/lpd.


4.0 Comandos Basicos
Para comenzar a dar vueltas por linux vayamos a la consola, es decir, daremos
intrucciones a traves de la linea de comando en modo texto. Si estas en KDE,
Gnome u otra interfaz, busca ejecutar Console o Terminal para comenzar a dar
comandos. Si tienes familiaridad con UNIX o BeOS esta seccion esta de mas para
ti. Ahora ante ti tienes un indicador o prompt que geralmente indica la
ubicacion actual y el nombre de usuario por el que logeaste junto al nombre
del host local. Veamos primero que nada donde estamos, esto se logra con el
comando pwd que nos mostrara el directorio actual

<root@localhost>#pwd
/root
<root@localhost>#


estamos en /root , la carpeta equivalente al /home de los demas usuarios.
Ahora vamos a a otro directorio, esto a traves del comando cd

<root@localhost>#cd /
<root@localhost>#


esto nos lleva a la raiz, ahora veamos que contiene esta raiz

<root@localhost>#ls
bin             home            proc            usr
boot            lib             root            var
dev             lost+found      sbin
etc             mnt             tmp


ahora, para ver el contenido de un directorio no es necesario tener que cambiar
la posicion actual e ir hacia el directorio con cd . Por ejemplo, veamos el
contenido de /root sin tener que salir de la raiz.

<root@localhost>#ls /root
Desktop         fye3.tgz        linux.txt       logofye.jpg
hentay.mpg      h001.jpg        h002.jpg        h003.jpg


Ahora que sabemos cambiar de directorio con cd y listar el contenido de los
mismos con ls, vamos un poco mas alla y creemos directorios y organizemos
nuestros archivos.

<root@localhost>#cd root
<root@localhost>#mkdir fye
<root@localhost>#mkdir hentay


Hemos creado los directorios fye y hentay dentro de /root, y aun cuando no es
necesario ir al directorio root, eso hicimos. Alternativamente podriamos haber
hecho lo siguiente directamente desde la raiz / sin necesidad de hacer cd root.

<root@localhost>#mkdir /root/fye
<root@localhost>#mkdir /root/hentay


Ahora copiaremos  los archivos hentay.mpg, h001.jpg, fye2.tgz, linux.txt y
logofye.jpg hacia los directorios que creamos, esto lo vamos a hacer
directamente desde la raiz, y sin cambiar hacia /root

<root@localhost>#cp /root/fye3.tgz /root/fye
<root@localhost>#cp /root/logofye.jpg /root/fye
<root@localhost>#cp /root/linux.txt /root/fye
<root@localhost>#cp /root/h* /root/hentay


Todo muy claro espero hasta ahora. Notese el ultimo comando en el que se
utiliza un comodin * para copiar asi todos los archivos que tengan en su
nombre por primera letra h. Ahora removeremos los archivos que quedaron en
/root y que ya copiamos, utilizando comodines para acortar y ahorrar tiempo.

<root@localhost>#rm /root/*.tgz
<root@localhost>#rm /root/h*
<root@localhost>#rm /root/l*


Digamos que nos arrepentimos y quermos volver a colocar los archivos que
copiamos hacia su antigua ubicacion en /root. Para evitarnos copiar y luego
eliminar simplemente moveremos los archivos.

<root@localhost>#mv /root/fye/* /root/
<root@localhost>#mv /root/hentay/* /root

Veamos como va todo, cambiando de la raiz al directorio /root

<root@localhost>#cd /root
<root@localhost>#ls -F
Desktop/        hentay/         fye/            h003.jpg
fye3.tgz        linux.txt       logofye.jpg
hentay.mpg      h001.jpg        h002.jpg


Utilizamos el modificador -F para que el listado diferencie con / entre
directorio y archivos individuales. Ahora eliminaremos los directorio hentay y
fye. Recuerda que aun estamos en /root.

<root@localhost>#rmdir fye
<root@localhost>#rmdir hentay
<root@localhost>#ls -F
Desktop         fye3.tgz        linux.txt       logofye.jpg
hentay.mpg      h001.jpg        h002.jpg        h003.jpg


Volvimos al inicio.

Vamos a seguir moviendonos, pero dentro del contenido de un disco flexible.
Como se vio anteriormente los dispositivos de sistema estan en /dev, mas estos
son archivos que hacen referencia a los recursos, por lo que no son accesibles
directamente. Para poder accesar un dispositivo de disco, debemos comunicarlo a
a nuestro sistema de archivos. Para eso montamos u unimos el sistema de
archivos del disquete dentro del sistema de archivos de linux, mediante el
comando mount. El que ara accesible el contenido del disco anadiendolo a un
directorio de nuestro sistema de archivos - el directorio seria el punto de
union o punto de montaje -mount point-. Creemos un directorio para esos fines,
dentro de /mnt solo para acerlo mas entendible o logico, ya que el punto de
montado puede ser cualquier directorio.

<root@localhost>#mkdir /mnt/floppy
<root@localhost>#mount /dev/fd0 -t vfat /mnt/floppy

Ahora podemos accesar al contenido del disquete desde /mnt/floppy.

+Sintaxis del comando mount
mount /dev/[dispotivo] -t tipo /[puntodemontaje]

Los dispotivos de disco en /dev quedaron explicados en la primera parte, el
punto de montaje es el directorio donde se quiere montar el disco para hacerlo
accesible, y el tipo que se indica despues del modificador -t corresponde al
tipo de sistema de archivos que utiliza el disco, en este caso es vfat pues es
un disco formateado en MS-DOS utilizando el sistema de archivos FAT.

Tabla de Tipos de Sistemas de Archivos Comunes
	ext2            Linux
	msdos           Microsoft DOS
	vfat            Windows 95
	fat32           Windows98/Windows2000
	iso9660         Utilizados por los CD-ROMS


Volviendo a donde estabamos, ya tenemos el disquete montado asi que veremos
que tiene

<root@localhost>#ls /mnt/floppy
linux2.txt              linux3.txt              linux4.txt
beos.txt                windowsnt.gif           command.com

Copiemos esto a /root utilizando multiples argumentos en el comando cp

<root@localhost>#cd /mnt/floppy
<root@localhost>#cp l* beos.txt windowsnt.gif /root

Copiamos todo menos command.com hacia /root. Ahora veremos que contiene los
archivos de texto, atraves del comando cat y more.
cat es el comando con el que es posible visualizar los archivos directamente,
listando el contenido en pantalla de forma inenterrumpida.
more encambio presenta el contenido de los archivos de forma pausada, ya que
el usuario debe presionar espacio para continuar la visualizacion. Muy util
para archivos largos. Digamos que al ver los archivos nos damos cuenta que
linux1.txt, linux2.txt, linux3.txt y linux4.txt son partes de un mismo texto y
estan divididos. Como unirlos? Aplicaremos el comando cat y un redireccionador
>

<root@localhost>#cd /root
<root@localhost>#cat linux1.txt linux2.txt linux3.txt linux4.txt >linux1234.txt

Si hicieramos un cat 12345.txt veriamos que contiene los 4 archivos, debido a
que redireccionamoes cat hacia un archivo en ves de hacia la consola
(pantalla) donde envia los datos de forma predeterminada. Entonces, envez 
de enviar un archivo tras otro a la pantalla, los tiro hacia un archivo 
gracias al redireccionador > que indicaba hacia linux1234.txt
Ahora  a~adiremos el texto de Beos (beos.txt) a linux1234.txt, esto tambien a
traves de redireccionadores, pero en este caso, >> ya que este no es
destrucctivo como > Esto es ya que > redirecciona y sobreescribe, mientras que
>> redirecciona anadiendo a un archivo existente sin sobreescribir los datos.

<root@localhost>#cat beos.txt >> linux1234.txt

Y finalmente a~adiremos un comentario final al archivo, utilizando echo,
comando que imprime texto de forma predeterminada en la pantalla, pero que
puede ser modificado si lo redireccionamos.

<root@localhost>#echo Articulo por NDR113 >> linux1234.txt

Asi quedara la ultima linea del archivo firmada.. XD

Este sistema operativo se define a si mismo como el primer intento por crear

Intentare analizar este OS de la forma mas simple posible, sin grandes detalles 
tecnicos y mas bien enfocado en sus aplicaciones presentes y futuras. 
Primero que nada BeOS en su edicion personal esta disponible para ser descargado 
desde http://free.beos.com , en un archivo ejecutable para Windows de 42 MB. 
El que al correrse instala en el tarro una imagen de disco de alrededor de

1. Iniciacion 
Demora entre 20 y 30 segundos en botear y cargarse -rapido, no?-. Si partes 
de un disco de inicio como yo, te tomara unos 50 segundos. 
En este tiempo se carga el kernel, se detecta el hardware y se inician los  
servicios.  
 
 
2. Interfaz 
La interzas de BeOS podria definirse como una cruza entre MacOS y KDE, 
debido a sus formas en extremo simplistas y funcionales, asi como la
utilizacion   de varios workspaces (areas de trabajo) -funcion ampliamente
utilizada en las interfazes para Linux-. 
En cuanto a el escritorio, en este estan presente los iconos de la papelera,  
algunos vinculos a informacion de BeOS tanto local como en Internet, y  
vinculos a los discos. 
Para accesar a las aplicaciones y funciones del sistema hay una peque~a barra  
de herramientas en la que tb se ven las aplicaciones actualmente en ejecucion,  
documentos y programas que fueron recientemene utilizados, preferencias de  
BeOS, etc. 
Bastante simple y facil de utilizar en resumen, con un buen uso del espacio, 
simplificacion de los controles de ventanas, y algunas peque~as mejoras como 
barras dezplasables en las que estan presentes los botones de subir y bajar  
en ambos extremos, ahorrando tiempo de mouse. 
 
 
3. Que hay bajo la interfaz 
La base del sistema es un kernel multihilos, preparado para procesamiento 
paralelo (soportando hasta 8 procesadores) y compatible con POSIX/Unix. 
Los procesos y aplicaciones corren bajo su propia asignacion de memoria
protegida,  con lo que se espera obtener mayor estabilidad en el trabajo
multitarea.  El sistema de archivos (BFS) es de 64 bits, con lo que soporta
teoricamente hasta  18 TeraBytes (18 mil Gb), ademas que tiene acceso a discos
o particiones  HFS (MacOS), FAT16/32, vFAT, ISO9660 (CDROM), Ext2 (Linux),
NTFS (WinNT) y  NFS (OS/2). 
La configuracion del hardware es completamente automatizada, lo que se debe 
en gran parte a que el hardware soportado es relativamente nuevo. BeOS no
ofrece  una gran compatibilidad hacia dispositivos antiguos, por lo que sus
ingenieros  pudieron empujar el sistema para rendimientos superiores sin
preocuparse por  este problema. Aun asi, el soporte es suficientemente amplio
para maquinas   relativamente nuevas... dudas? puedes obtener la lista BeOS
Ready en   http://www.be.com/products/freebeos/beosreadylist.html 
En cuanto a los controladores del hardware, estos corren mas bien como
servicios  que pueden ser detenidos o iniciados segun el OS lo determine,
liberando  recursos al descargar de la memoria los drivers de dispotivos que
no esten   en uso.

Al contrario de Linux, BeOS no es un sistema operativo dise~ado para tareas

Se pueden configurar las siguientes opciones en BeOS > Preferences > Network 
 
Domain Name     Nombre del dominio al cual pertence la maquina 
Host Name       Nombre del Host 
Primary DNS     Servidor DNS 1 
Secundary DNS   Servidor DNS 2 
 
Por el momento el soporte a NICs (Tarjetas de Interfaz de Red), es bastante

protocolos TCP/IP y los servicios AppleTalk.  
Finalmente, se deben dar los siguientes valores para el acceso a la red local 
 
IP: Puede ser obtenido de un servidor DHCP de forma automatica, o,

Gateway.

de usuario, la password y los valores del servidor: 
 
IP:             Asignado o Fijo 
Primary DNS:    Servidor DNS 1  
Secundary DNS:  Servidor DNS 2 
Server Type:    PPP o Unix 
 
En cuanto al soporte de modems, BeOS incluye secuencias de iniciacion y
comandos  AT para los modems mas populares, e incluso soporta algunos
softmodems -winmodems-,  asi como modems basados en DSPs. Aun asi no podria
asegurar cuan bueno es  este soporte por cuanto no lo he probado, y

BeOS se perfila como un sistema operativo personal, con enfasis en la

Aun asi no es todo maravilloso, pues BeOS tiene ciertas limitantes y podria
