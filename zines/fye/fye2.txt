  Hola a todos los lectores de FYE de parte del NeGrO...en esta edicion vamos
a proseguir con este peke¤o curso de C ke empezamos en el primer numero de FYE
asi ke preparense...
  En el capitulo anterior kedamos en lo de las funciones y todo eso...asi ke
ahora vamos a explicar algunos comandos interesantes...
  Voy a explicar algunas funciones ke son de relevancia tomandolos a partir
de un ejemplo nada de util...

#include <stdio.h>   /* incluimos la biblioteca stdio */
main()               /* llamando a main,desde aki se ejecuta el programa */
{
 int numero,limite; /* declaramos las variables como enteros */
 limite=10;         /* le damos el valor 10 a limite */ 
 numero=0;          /* y 0 a numero */
 while(numero<limite){    /* aka dice: mientras numero menor a limite */
        printf("NEGRO\n"); /* entonces escribir NEGRO en una nueva linea */
        ++numero;          /* luego incrementar en uno a la variable numero */
 }            /* ya ke numero es mayor o igual a limite terminanos el while */ 
}             /* fin del programa */

  Bueno creo ke el ejemplo no puede ser mas inutil,pero sirve para mostrarnos
como operan algunos comandos...asi ke leyendo atentamente las acotaciones
ke inclui al lado de kada linea creo ke ya deberian tener una idea de ke
significa cada cosa...
  Este programa lo ke hace es escribir 10 veces NEGRO,una vez por linea y
luego termina...es un programa extremadamente sencillo ya ke no tiene ninguna
dificultad para entenderlo.
  Explikemos...
  #include <stdio.h>  aka llamamos a stdio.h
  main(){             aka iniciamos main
  int numero,limite;  aka estamos declarando las variables...en C todas las
                      variables deben declararse y expresar a ke tipo de ellas
                      corresponden...por ej:
                      øint = entero (16 o 32 bits)
                      øfloat = punto flotante (32 bits)
                      øchar = caracter (un solo byte)
                      øshort = entero corto
                      ølong = entero largo
                      ødouble = flotante de doble precision
                       despues de terminar la declaracion se agrega un punto
                       y coma,esto se hace en todas las proposiciones individuales
                       Debo indicar ke el hecho de ke esten las dos variables
                       declaradas con un solo int no kiere decir ke no puedan
                       declararse por separado.
  limite=10;           Estas son las proposiciones de asignacion donde damos
  numero=0;            a las variables sus valores iniciales...ya ke nuestro
                       objetivo es ke se escriba 10 veces NEGRO,declaramos
                       a numero como cero.
  while(numero<limite){   aka iniciamos el ciclo while,lo ke hace esta funcion
                          es probar la condicion ke esta entre parentesis (sus
                          argumentos) y luego ejecutar el cuerpo del ciclo (lo
                          ke esta dentro de sus {}...despues de ejecutar el
                          ciclo,vuelve a probar la condicion,si esta se hace
                          falsa,el ciclo continua y sigue con el sig. comando,
                          si el ciclo es verdadero vuelve a ejecutarse...
  printf("NEGRO\n");   Aka escribimos en la pantalla la palabra NEGRO en una
                       nueva linea
  ++numero;            ++ es el operador ke incrementa en uno,tambien podria
                       haber sido numero=numero+1 pero ciertamente ++ es mucho
                       mas comodo.Tambien existe un ekivalente ke decrementa
                       en uno (--).++ y -- pueden ir antes o despues de su
                       variable,en ambos casos aumentan o disminuyen en uno
                       pero estos tiene usos diferentes...++ como prefijo
                       incrementa a la variable antes ke el valor se use,mientras
                       ke ++ como postfijo lo hace despues de ke el valor de la
                       variable sea empleado.
  }                    cierra el cuerpo del ciclo while
  }                    cierra las proposiciones de main por lo tanto el programa.

  Ke simple no???...bueno pues esto les sera muy util para empezar a analizar
algunos codigos fuentes de programas ke andan por ahi (sniffers,zapper,crack,
etc,etc),obviamente no entenderan todo pero ya tienen la base :)

  Ahora vamos a a¤adir una linea a este programa para ver un caracteristica
bastante util del printf...

#include <stdio.h>   
main()               
{
 int numero,limite; 
 limite=10;          
 numero=0;          
 while(numero<limite){    
        printf("NEGRO\n"); 
        ++numero;          
 }             
 printf("He escrito %d veces NEGRO\n",numero); /*Aka decimos las veces ke se escribio NEGRO*/
}

  No voy a explicar una por una las lineas porke ya lo hice...pero miremos
la penultima linea...aka usamos un printf y le hacemos decirnos las veces ke
escribio NEGRO,la forma en ke lo hace es muy simple,para cada %(algo) nos
imprime el valor de una variable ke esta indicada segun va primera o segunda
despues de finalizado el mensaje ke sera impreso,o sea luego de la coma...
en este caso para nosotros,nos imprime el valor de la variable numero.
  Para hacerlo mas claro escribire otro ejemplo...

#include <stdio.h>
 main(){
 int uno,dos;
 uno=1;
 dos=2;
        printf("Los numero son %d y %d\n",uno,dos);
 }

  Como ven el primer %d nos mostrara el valor de uno y el segundo el valor de
dos... asi de simple :) ...
  El indicador d nos indica ke la variable es un entero...aca les muestro
todo de una vez mejor.

ød      entero decimal int
øi      entero int ke puede estar en octal o hexadecimal
øo      entero octal int
øu      entero decimal sin signo (unsigned int)
øx      entero hexadecimal int
øc      caracter char
øs      cadenada de caracteres char
øe,f,g  numero de punto flotante,punto decimal y exponente optativos;float o double
øl,h    enteros largo y corto respectivamente;long,short
%       literal (nos muestra un %)

  Tambien tenemos las opciones de darle formato al numero...por ejemplo si
nuestra variable era un float,y keremos ke salga con dos decimales,hacemos
esto:

 printf("El numero es %.2f\n",variablefloat);

  Y si keremos ke sea sin decimales...

 printf("El numero es %.0f\n",variablefloat);

  Podemos tambien darle 6 caracteres de ancho para ke ocupe...empezando la
variable a usarlos a partir de la izquierda...

 printf("El numero es %6f\n",variablefloat);

  Ahora los 6 caracteres de ancho mas dos de decimal...

 printf("El numero es %6.2f\n",variablefloat);

  Ahora los mismos 6 caracteres de ancho mas dos de decimal,pero empezando
desde la izquierda

 printf("El numero es %-6.2f\n",variablefloat);

  Bueno creo ke ya entendieron...
  Probemos ahora a darle mas formato al texto...ke tal si le agregamos un
tabulador entre medio de dos variables...

 printf("Los numeros son %.0f\t%d\n",variablefloat,variableint);

  Entre la impresion de la variable float y la int hay una tabulacion de
separacion...
  Creo ke seria util dar a conocer el conjunto completo de secuencias de
escape de printf...

  \a caracter de alarma
  \b retroceso
  \f avance de hoja
  \n nueva linea
  \r regreso de carro
  \t tabulador horizontal
  \v tabulador vertical
  \\ diagonal invertida
  \? interrogacion
  \ï apostrofo
  \" comillas
 \ooo numero octal (0...7)
 \xhh numero hexadecimal (0..9,a..f,A..F)

  Bueno creo ke ya kedo todo entendido en cuanto a los ejemplos ke di...ahora
pasemos a analizar otra cosa.
  Lo ke me gusta de los lenguajes de programacion,es ke se puede escribir un
mismo codigo de muchas maneras distintas,por ejemplo,si yo hubiera kerido
escribir el codigo del programa NEGRO con la instruccion FOR en vez de WHILE se
habria visto asi:

#include <stdio.h>
main(){
   int numero,limite;
   limite=10;
   for(numero=0;numero<limite;++numero)
   printf("NEGRO\n");
}
  Algo mas corto el codigo pero con el mismo resultado...El limite inferior
y su comparacion con el superior ahora son partes de los argumentos de FOR.
La proposicion FOR es un ciclo igual ke WHILE pero mas generalizado...dentro
de los parentesis hay tres secciones separadas por punto y coma,la primera
se ejecuta una sola vez antes de entrar al ciclo en si,la segunda es la ke
controla el ciclo,o sea la condicion de prueba,cuando esta se hace falsa el
ciclo se termina,y la tercera es el incremento de avance (o sea despues de
ejecutarse la tercera vuelve a ejecutarse la segunda hasta ke se haga falsa).
  Realmente simple como pueden ver...es importante aclarar ke en este caso
el cuerpo del ciclo no va entre {} porke es una sola proposicion individual
la ke se ejecuta (o sea printf("NEGRO\n");)...esto tambien se puede aplicar
a WHILE,veamos un ejemplo de este tipo:

 while(Zweastic=enfermo)
 printf("El Zweastic es un enfermo\n"); /* este ciclo seria infinito XDDD */

y listo,no necesitamos {},cuando el cuerpo tiene mas de una proposicion deben
encerrarse entre {}.

 Mirando el codigo de nuestro programa de una forma critica podemos preguntar:
¨¨ Porque se declaro la variable limite cuando habria sido tan facil escribir
como condicion de prueba en el ciclo while numero<10 ??
  La respuesta es porke en C no es muy bueno poner entre medio del codigo
numeros ke no nos indican nada de nada (se les llama "numeros magicos"),ke
kiere decir esto:supongamos ke hicimos el feroz programa ke tiene un monton
de numeros ke indican algunas cosas dentro del codigo pero ke para el ke lo
lee no significan nada,esto a la larga del tiempo puede resultar una desventaja
incluso para el propio programador pues puede olvidar el porke de cada "numero
magico",entonces para evitar esto le asignamos una variable a cada numero ke
utilizamos.
  Ahora talves piensen:"Beo ke zi,haora conpremdo ezhto,er codigo ta vien
ezkryto"...pero esto no es asi...el codigo no esta mal...pero tampoco esta
bien,veamos porke:
  En C existe una forma de tratar a los numeros magicos,y es dandoles nombres
significativos.Esto se hace con la linea DEFINE,ej:

#define nombre valor_o_texto_de_reemplazo

  Gracias a esto,cada vez ke se invoke a nombre se reemplazara por lo ke le
indicamos a DEFINE.
  Veamos como se veria el codigo con un DEFINE:

#include <stdio.h>
#define LIMITE 10
main(){
   int numero;
   for(numero=0;numero<LIMITE;++numero)
   printf("NEGRO\n");
}

...ke tremendo cambio...en realidad es tan correcto como la anterior forma,
pero ahora arreglamos el codigo para ke sea mas legible (obviamente de la
forma ke miremos este codigo es entendible,pero los DEFINE son muy buenas
ayudas en codigos mas largos).Los nombres de constantes simbolicas (las
declaradas con un DEFINE) se escriben con mayusculas por convencion (yo no
asisti...no me invitaron :(  ).Noten ke una linea DEFINE no termina con punto
y coma,y se declara antes ke la funcion main.

   Sigamos avanzando en conocimientos de C...ahora veamos este codigo:

#include <stdio.h>
#define LIMITE 10
main(){
int numero;
for(numero=0;numero<LIMITE;++numero){
	printf("NEGRO\n");
	}
if(numero==10)
	printf("Fueron %d NEGROs\n",numero);
else if(numero!=10)
	printf("Se imprimio %d veces NEGRO\n",numero);
}
   JAJAJAJA...en ningun otro lado van a encontrar un codigo mas inutil ke este
puesto ke incluye operaciones ke no tienen razon de ser...PERO!!!...todo esta
planificado con un fin...jeje.
   Como veran, en este codigo he incluido las proposiciones IF y ELSE IF,y
cual es la mision de estas?? la de decidir la respuesta correcta a la hora
de decirnos cuantas veces se imprimio IF...este codigo no tiene nada de bueno,
en realidad no esta malo,si tu lo compilas ejecutara perfectamente,pero es
ke utilizo algunas cosas de una forma ke no deberia o sea sin razon...como ya
dije esto tiene un proposito...lo ke yo kiero lograr con este codigo es ke
se aventuren a cambiar el codigo,modificarlo y todo eso para ver ke respuesta
obtendran.Por ejemplo,si yo no cambio nada la respuesta seran 10 NEGROs y la
frase "Fueron 10 NEGROs",pero si yo cambio el LIMITE a 9,obtendre como
respuesta 9 NEGROs y la frase "Se imprimio 9 veces NEGRO"...asi ke como ven
segun el codigo la respuesta sera diferente...es un programa indeciso..jeje :)
Tambien resultaria interesante ke experimentaran cambiambo el valor de la
variable numero o la condicion de ejecucion del ciclo,u otra cosa...mi idea
es ke se atrevan a experimentar para ver ke respuestas obtienen o ke errores
reportara el compilador por akellas modificaciones invalidas.
  Observemos el codigo:hasta la linea siete todo es normal,pero luego entra en
accion un IF,este prueba la condicion de ke la variable numero sea igual a
10 (la prueba de igualdad se hace con == no =,ya ke = se utiliza para asignar
un valor a una variable y lo ke estamos haciendo aka es ver si esa variable
tiene ese valor,es importante fijarse en este detalle;tambien vemos en este
codigo la presencia de != ke significa "diferente a"),si es cierta ejecuta
el printf,si no es cierta pasa a ejecutarse su alternativa ke esta
representada por el ELSE IF,este tambien prueba su condicion y ya ke la
primera es falsa esta necesariamente es verdadera.Creo ke con esta explicacion
ya entendieron la forma de operar de IF pero hagamoslo mas claramente:

  IF (expresion1)
     proposicion1
  ELSE IF (expresion2)
     proposicion2
  ..
  ELSE IF (expresionM)
     proposicionM
  ..
  ELSE
     proposicionN

  Como pueden ver aka esta bien empleado el IF-ELSE.Lo primero ke se hace
es probar la condicion del IF,si esta es falsa,se pasa a la del sig. ELSE IF,
hasta encontrar la ke sea verdadera,si ninguna lo es,se ejecutara el ultimo
ELSE (el cual no lleva IF) y ejecutara su proposicion...Si entendieron esto
creo ke ya sabran ke la forma correcta de haber escrito el codigo anterior
era asi:

#include <stdio.h>
#define LIMITE 10
main(){
int numero;
for(numero=0;numero<LIMITE;++numero){
	printf("NEGRO\n");
	}
if(numero==10)
	printf("Fueron %d NEGROs\n",numero);
else    /* notese ke aka eliminamos la condicion puesto ke ya sabemos ke no es 10 */
	printf("Se imprimio %d veces NEGRO\n",numero);
}

  Si en el primer codigo de este programa hubieramos cambiado la proposicion
del ELSE IF por...por numero==9,por ejemplo,no nos habria dado respuesta
alguna acerca de cuantas veces imprimimos NEGRO ya ke ninguna de las dos
condiciones se hizo verdadera por lo tanto ninguna es valida para ejecutarse.
El ELSE solitario tiene una razon de ser como pueden ver,y sirve para ke,
como en caso de nuestro programa,ejecute la opcion ke inevitablemente se
cumple...veamoslo asi:

   En el primer codigo pusimos else if(numero!=10) lo ke significa ke si
   el numero es distinto a 10 la frase nos dara distinta con respecto a
   si el numero es 10 (eleji 10 por ser el limite,no es ke se me haya
   antojado asi...bueno en realidad si se me dio la gana asi,pero porke
   era el limite lo hize con el 10).Esta linea de codigo esta escrita de mas
   ya ke nos da la respuesta para la condicion ke se cumple inevitablemente
   con respecto a la primera o IF.Asi ke es por eso ke en vez de usar el
   ELSE IF,en este caso es mas CONVENIENTE usar solo ELSE,por motivos de
   mayor comprension a la hora de mirar el codigo.

  Con esto les demostre una maxima ke tienen los programadores y ke es algo
como: "Si existe una forma de hacer una cosa,siempre existira otra forma mas
complicada de hacerla",no es exactamente asi,pero esa es la idea (creo).
